\subsection{Network Layer Architecture}

The network layer constitutes the control plane of the proposed LoRa-based V2V mesh system and 
is designed as a portable, hardware-independent core positioned between application services and radio-facing 
transmission services. Its architectural role is to transform locally generated safety data into 
dissemination-ready network frames, to evaluate received frames against propagation policy, and to 
regulate rebroadcast behavior so that dissemination reliability is improved without incurring unnecessary 
channel occupation. This role is realized through explicit boundary abstractions that isolate forwarding 
intelligence from platform-specific communication and sensing mechanisms, thereby allowing identical network 
behavior to be preserved across embedded deployments and host-side simulation environments.

At the transmission boundary, each outgoing frame is structured as a fixed network header followed by 
application payload bytes. The header encodes message identity, temporal validity, origin context, per-hop 
transmitter context, and propagation constraints, so that downstream nodes can make forwarding decisions 
from self-contained frame metadata. The fixed header length establishes deterministic parsing and bounded 
overhead, while the payload limit is derived directly from the LoRa frame-size ceiling to guarantee that 
all generated frames remain admissible for the underlying physical channel. Within this model, a unique 
message identifier combines source identity with a local sequence progression, enabling replay suppression 
and idempotent handling throughout multi-hop dissemination.

The layer distinguishes immutable origin metadata from mutable per-hop transmitter metadata as a central 
design principle. Origin metadata remains unchanged for the lifetime of a message and represents the initial 
source state at creation time. By contrast, transmitter metadata is refreshed at every relay and represents 
the most recent forwarding node. This semantic separation enables the routing process to reason simultaneously 
about global propagation intent and local progression, which is essential for both hold-back scheduling and 
redundancy suppression in dynamic vehicular topologies.

Operationally, the data plane follows two coordinated paths: synchronous origination and asynchronous 
reception-driven forwarding. During origination, the layer validates payload admissibility, assembles the 
network header using local temporal and mobility context, registers the message identity in a duplicate-tracking 
structure, and emits a single broadcast frame. During reception, incoming frames are first buffered through 
a bounded event queue and then processed by worker execution contexts that perform parse, policy evaluation, 
local delivery, and conditional relay scheduling. This decoupled architecture prevents receive callbacks from 
becoming policy execution bottlenecks and improves temporal predictability under bursty traffic. The receive-path
processing sequence is shown in Fig.~\ref{fig:network-layer-sequence}.

Forwarding decisions are produced by an ordered evaluation pipeline that prioritizes safety and bounded 
propagation. Frames are first screened for temporal expiry, replay status, and residual hop budget, 
followed by checks on distance-constrained dissemination and directional eligibility when directional 
propagation is requested. Messages that satisfy these gates are considered relay candidates and assigned 
a deferred forwarding delay rather than immediate retransmission. The delay is computed from a composite 
estimate of path progress and link quality, and then modulated by traffic priority so that urgent safety 
traffic propagates with lower latency while less critical traffic is naturally delayed. This mechanism 
implemented contention-aware spatial diversity by encouraging favorable relays to transmit earlier and 
discouraging simultaneous redundant rebroadcasts. The ordered decision logic and resulting outcomes are
summarized in Fig.~\ref{fig:network-layer-decision}.

To further limit broadcast redundancy, the architecture incorporates two complementary suppression mechanisms. 
First, duplicate filtering prevents repeated processing and forwarding of already observed message identities. 
Second, queued relays can be canceled when subsequently overheard transmissions indicate that upstream 
progression had already covered the local node's prospective forwarding contribution. This geometric suppression 
strategy reduces superfluous retransmissions without requiring explicit coordination exchanges, which is 
advantageous in bandwidth-constrained and intermittently connected vehicular channels.

Concurrency management is integral to architectural correctness. The network layer employs separate execution 
contexts for receive-path processing and timer-driven relay emission, with mutex- and condition-variable-based 
synchronization around bounded shared queues and callback handoff state. Lifecycle control is designed to be 
robust under repeated start-stop operation, ensuring deterministic initialization, safe termination of background 
activity, and clean release of receive handlers during shutdown. This threading model provided clear separation 
between ingress handling, routing computation, and deferred transmission timing.

The architecture also exposes bounded configuration controls that govern duplicate-cache capacity, relay-queue 
capacity, receive-queue depth, and forwarding delay limits. These parameters defined the operational envelope 
for memory consumption, latency, and admission behavior, and therefore enabled systematic trade-off analysis 
between responsiveness and channel load. In combination with fixed frame-level constraints, the configuration 
model promotes predictable runtime behavior across heterogeneous hardware targets.

Finally, the same network-layer core is embedded in a deterministic simulation runtime to support repeatable 
validation under controlled mobility and channel conditions. Each simulated node instantiates the same logical 
network functionality as a deployed node, while a shared virtual clock and event-driven channel model governs 
transmission timing, reception outcomes, and channel impairments. Because routing and forwarding semantics are 
preserved across deployment and simulation, the architecture supports direct transfer of behavioral findings 
from laboratory evaluation to field-oriented configurations.
