\subsection{Link Layer Architecture}

The link-layer architecture operationalizes the communication semantics required by the upper stack while 
preserving strict boundaries with routing and application logic. Its principal responsibility is to provide 
a dependable single-hop delivery substrate over a constrained long-range channel, including frame construction, 
channel access coordination, reception management, and controlled energy behavior. In this design, upper layers 
submitted payloads and consumed delivery events through an asynchronous service interface, whereas all 
radio-facing behavior remained encapsulated within the link layer. This separation ensured that forwarding 
policy, topology reasoning, and service semantics could evolve independently of modem-specific constraints.

The wire format is organized around a compact fixed-size header followed by a bounded payload region, 
thereby ensuring deterministic parsing cost and explicit interoperability guarantees across nodes. Header 
fields encode source and destination identifiers, message classification, sequence numbering, control flags, 
and payload length. The addressing model used node-local unicast identifiers and a dedicated broadcast 
destination value for one-to-many dissemination. Sequence and flag metadata jointly support reliability 
signaling without imposing mandatory overhead on all traffic classes. A reserved message type is allocated 
for acknowledgment control traffic, preventing ambiguity between reliability signaling and application-defined 
data categories.

Medium access control follows a listen-before-send strategy based on channel activity detection. Prior to 
transmission, the transmitter assesses channel occupancy and defers when LoRa activity is detected, 
reducing collision probability under shared medium contention. When repeated busy indications occurred, 
bounded retry behavior with randomized backoff is applied to decorrelate concurrent transmitters and limit 
persistent synchronization effects. If the medium remains unavailable beyond the configured sensing budget, 
the pending transmission is discarded to prevent unbounded latency growth and queue destabilization. 
This policy favored predictable temporal behavior and spectrum discipline over aggressive persistence. The
corresponding runtime state transitions are shown in Fig.~\ref{fig:link-layer-state-machine}.

The transmission workflow combines queued frame dispatch with optional per-frame reliability. Once admitted 
for transmission, a frame undergoes channel sensing and, when clear, is emitted over the radio interface. 
For traffic that does not request reliability, completion ends at transmit confirmation. For reliability-enabled 
traffic, the sender enters a bounded acknowledgment wait phase and validates incoming acknowledgments against 
sequence and addressing context. Successful matches terminate the transaction with positive outcome signaling 
to the upper layer. Timeout events trigger controlled retransmission attempts up to a configured maximum, 
after which the frame is reported as undelivered. This selective reliability model allows critical safety 
messages to request stronger delivery assurance while preserving lower overhead for noncritical traffic, as 
illustrated in Fig.~\ref{fig:link-layer-sequence}.

Inbound processing follows a strict filtering pipeline to protect upper-layer logic from malformed or 
irrelevant traffic. Frames below the minimum valid length are rejected immediately. Valid frames are parsed, 
and link-quality observations are recorded for neighbor awareness. Destination filtering then removes packets 
not addressed to the local node or broadcast group. Pure acknowledgment frames are consumed internally by the 
reliability subsystem and not exposed as application payload events. When a received data frame requested 
acknowledgment, a corresponding acknowledgment response is generated at link level. Only after these checks 
are satisfied is payload data delivered upward, ensuring that upper layers observe semantically meaningful 
and policy-compliant traffic.

Energy management is integrated into runtime behavior through adaptive receive-idle control. 
The receiver remains active to preserve responsiveness but transitions to a low-power modem state 
after sustained inactivity. Wake-up is event-driven: outbound traffic demand or relevant radio interrupt 
activity restores active operation and resumes normal processing. This approach balances readiness and battery 
efficiency, a necessary trade-off for vehicular platforms that must sustain long operating intervals without 
sacrificing timely communication when events occur.

Hardware abstraction is achieved through a backend interface that isolates protocol and state logic from 
concrete transceiver drivers. The link layer relies on this contract for channel sensing, transmission, 
reception, signal-quality observation, interrupt signaling, and power-state transitions. As a consequence, 
the same link-layer behavior can execute against production radio hardware or controlled test backends 
without architectural change. This design improves portability, enables deterministic validation under 
laboratory conditions, and reduces coupling between communication policy and platform-specific implementation 
details.
