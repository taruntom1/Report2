\subsection{Top-level Overview}

The proposed LoRa-based V2V system follows a \textbf{three-layer architecture} that separates 
application concerns, network intelligence, and radio/link execution.

At a system level, the architecture is designed to support \textbf{distributed safety message dissemination} 
among vehicles over constrained LoRa links, while preserving modularity and testability.

\begin{itemize}
    \item The \textbf{Application Layer} generates outbound vehicle messages (currently test messages) 
    and consumes delivered inbound messages.
    \item The \textbf{Network Layer} acts as the system's control core: it encapsulates application 
    payloads with network metadata, decides whether packets should be delivered and/or forwarded, suppresses duplicates, and manages relay timing.
    \item The \textbf{Link Layer} provides LoRa frame transmission/reception over SX1278 hardware 
    (or equivalent backend), including packetization, radio task/state-machine execution, 
    channel sensing (CAD), and optional ACK/retry behavior.
\end{itemize}

In operation, application payloads flow downward (Application $\rightarrow$ Network $\rightarrow$ Link) 
for transmission, while received LoRa frames flow upward (Link $\rightarrow$ Network $\rightarrow$ Application).

The Network Layer is explicitly decoupled from hardware via an abstract link interface 
(\texttt{ILinkLayer}), enabling both real-radio and simulation-backed deployments.
