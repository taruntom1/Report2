\section{Techniques}
Forwarding decisions follow an ordered evaluation pipeline so that safety and bounded propagation are handled
first. We start by screening frames for temporal expiry, replay status, and the remaining hop budget. After
that, we apply the checks for distance-constrained dissemination and (if enabled) directional eligibility.

Medium access control uses a listen-before-send strategy based on channel activity detection. Before
transmitting, the node checks whether the channel seems busy and defers if LoRa activity is detected. This
helps reduce collisions on a shared medium.

We also use a selective reliability model. Critical safety messages can request stronger delivery assurance,
while noncritical traffic keeps lower overhead.
